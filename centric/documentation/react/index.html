Main folders explanation

app/

This folder contains all assets necessary to build the application (images, fonts, etc) and compiled files. This files are not ready to deploy on your server and they need to be compiled first.

  • components/ Here you will find all styles, scripts and JSX source files for the template.
  • fonts/ This folder contains all font files used in the app (except Google Fonts)
  • img/ This folder contains all images used in the app

  • server/ This folder contains server side files used for demonstration. In a real app your server should listen to specific address and return the content of this files (and/or others) with specific information.

  • bower_components/ This folder contains the vendor files used to include plugins and other components. This folder is managed using Bower, optionally, you can remove or upgrade the vendor file using such tool.

  • node_modules/ This contains also vendor assets used in the application and others required to build the application.

Main Files

In the root folder you will find the following files

  • package.json: contains all configuration for the npm packages
  • webpack.common.js: WebPack common configuration (shared between development and production)
  • webpack.dev.js: WebPack specific configuration for development environment
  • webpack.prod.js: WebPack specific configuration for production environment

Build

Important You only need to follow this instructions if you plan to use build tools.

Build Tools

Node.js is a platform built on Chrome’s JavaScript runtime for easily building fast, scalable network applications.

Webpack is a module bundler. It packs CommonJs/AMD modules i. e. for the browser. Allows to split your codebase into multiple bundles.

Bower is a dependency manager, it works by fetching and installing packages from all over, taking care of hunting, finding, downloading, and saving the stuff you’re looking for. Bower keeps track of these packages in a manifest file, bower.json.

The package includes under the root folder the file webpack.*.js, package.json and bower.json to install the required components in order to compile the source files.

Installing tools

The following steps are intended to be an orientation guide, if you are not experienced with this you will need to learn a bit more about it from Google :)

  • To install node and npm, go to http://nodejs.org/
  • Download and install GIT for your platform from http://git-scm.com/downloads

Once you have all tools installed

  • Move to the package root folder and run the following commands

    • This command will install all node dependencies. npm install.
    • This command will install vendor dependencies bower install

  • Finally run npm run start to start the application.

If everything goes fine, you should see some debug messages in your terminal. Webpack will start a server, open the browser and watch for file changes to compile them automatically.

Webpack configuration

If you need to change the configuration used by Webpack, mainly paths, you can do that by editing the files webpack.*.js according to the enviroment you want to change.

Main NPM commands

This command will build the app and open the browser

> npm run start

Delete build files

> npm run clean

This task will compile the application for distribution

> npm run build

Note When running in development mode webpack-dev-server creates a memory file system to place all compiled files
so you won't see files created in your disk.

Usage

To use his template was is strictly necessary that you use the build tools provided in order to compile source files for browsers to be able to understand them. You don't need nothing special more than just run the provided commands, webpack does all the job to create necessary files to serve the app.

We encourage you to use build tools, the benefits greatly outweigh the costs of learning.

The entry point of the application is the file app/App.jsx
In this files we declare all routes and import all necessary reference to each component.

The main component you must know is the Core component whihc makes possible to create the entire application structure
The following line includes that component:

import Core from './components/Core/Core';

This component also imports the following files 

// app/components/Core/Core.jsx

import Header from '../Header/Header';
import HeaderSearch from '../Header/HeaderSearch';
import Sidebar from '../Sidebar/Sidebar';
import Settings from '../Settings/Settings';

As you can see, by default the import statement consider the JSX extension to loads files.

Index

The file app/index.html is the main file that is loaded by the browser to start the application.
Actually, this file is treated as a template by webpack in order to generate the final index.html using the HtmlWebpackPlugin plugin. 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Centric + ReactJS</title>
</head>
<body class="theme-1">

    <!-- Application -->
    <div id="app"></div>

    <!-- GoogleMaps -->
    <script type="text/javascript" src="http://maps.google.com/maps/api/js?key=..."></script>

</body>
</html>

Note that there's no script and styles added. That's because we use a webpack plugin that inject dynamically all files generated for the application bundle.

Styles

Each component contains its own styles. Webpack read all folder recursively and includes theme into the final bundle when it sees any .scss file imported.

Scripts

For pure JavaScript files we add the .run suffix to the filename to make theme different from the component entry point. This way we can write specific JS that perform actions over such component but it doesn't interfere with the main file.

For example, the Dashboard components contains the following files

+---Dashboard
    +---Dashboard.jsx
    \---Dashboard.run.jsx

And then from Dashboard.jsx we import the JS file to run the main function. Note we use the method componentDidMount to make sure the script run after the component has been mounted so jQuery invocation can find the DOM elements.

// Dashboard.jsx
import DashboardRun from './Dashboard.run';

class Dashboard extends React.Component {

    componentDidMount() {
        DashboardRun();
    }
    ....

// Dashboard.run.js
function initDashboard() {
    ....
}

export default initDashboard;
Views (JSX)

Views are written in JSX format which a language similar to HTML markup but that also allows to embed JavaScript directly into the code. JSX is a statically-typed, object-oriented programming language designed to run on modern web browsers.

Routes

Routes are defined in file app/App.jsx and uses the react-router markup
Example:

ReactDOM.render(
    <Router history={browserHistory}>
        <Route path="/" component={Core}>

            {/* Default route*/}
            <IndexRoute component={Dashboard} />

            <Route path="dashboard" component={Dashboard} />
            <Route path="cards" component={Cards} />

            <Route path="charts">
                <Route path="radial" component={Charts.Radial} />
    ....

Ripple Effect

The ripple can be achieved on almost any component by adding the class .ripple

This class is handled by a script that display a simple animation in the middle of the element where it's attached.

If you want to customized the styles of scripts of this effect, you find all related files under components/ripple

Translation

The translation system uses jQuery Localize This modules simplifies the translation system by loading translate references from a JSON file and replacing the content where the reference has been used.

Examples

<small data-localize="dashboard.WELCOME"></small>

The JSON files with translation references are located in the folder app/i18n

Example content of the file app/server/i18n/en.json

{
  "WELCOME": "Welcome",
  "header": {
    "SEARCH": "Search.."
  }
}

As a convention, we use the key name with the translated text in uppercase.

Settings and Themes

All settings related files belongs to components/settings

There you will find the initial values for the different settings supported. The template markup for the settings panel (the right sidebar) and the styles for such panel.

Themes

Theming is based on class-names that provides colors to components.

All themes classes available are configured in settings.js

var themes = [
    'theme-1',
    'theme-2',
    'theme-3',
    'theme-4',
    'theme-5',
    'theme-6',
    'theme-7',
    'theme-8',
    'theme-9',
]

Each of this theme, named theme-* is applied to the body tag via JavaScript.

You can find the source of the themes definition in file app/components/common/themes.less

Default theme

To set a theme as default, just add the classname of the theme you want to the body tag in file app/index.html
Example: 

<body class="theme-1">
