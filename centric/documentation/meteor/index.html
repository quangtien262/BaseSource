Introduction

Getting started tips

  • Do not start from scratch, use an existing asset and modify it to learn how it works.
  • Explore the sources for ideas and sample code.
  • Use Firebug or Chrome Developer Tools to find bugs on your website. Using one of those tools will help you to save time analyzing the site and finding elements structure, like classes, id or tags
    • Quick tip: open your site with Chrome, press F12 and go to console tab, reload your page and if something goes wrong you will see your page errors in red text.
  • In case of errors, someone might have seen it too, you can try a Google search for a quick fix.

Starting the app

Download MeteorJS

Start downloading METEOR framework for your platform at https://www.meteor.com/install

Start the application

Run the following command in the meteor project root folder

npm install

meteor

Open your browser*

If everything goes fine, your application should be ready at http://localhost:3000/

Main folders explanation

The structure we have followed is pretty similar to the stated at the official documentation. Except for some parts we have omitted (just for simplicity), you can follow the official guide to learn more in deep how to structure your final app.
Please check it here http://guide.meteor.com/structure.html

client/

Files placed here are the entry point for the client side code of the application.

imports/

This files are imported from another files. By itself, those files are loaded by Meteor

pubic/

Everything placed here is for public access, like images, fonts, etc.

server/

Files placed here are the entry point for the server side code of the application.

Layouts

Main layout can be found in file imports/ui/components/core/core.layout.html with the following content:

<template name="layout">

    <div class="layout-container">

        <!-- header -->
        <header class="header-container">
            {{> header }}
        </header>

        <!-- Sidebar -->
        <aside class="sidebar-container">
            {{> sidebar }}
        </aside>
        <div class="sidebar-layout-obfuscator"></div>

        <!-- Main section -->
        <main class="main-container">

            <!-- Page content -->
            <section>
                {{> yield }}
            </section>
            <!-- Page footer -->
            <footer>
                <span>2016 - Centric app.</span>
            </footer>
        </main>

    </div>

    <!-- Search template -->
    {{> headerSearch }}
    <!-- End Search template -->
    <!-- Settings template -->
    {{> settings }}
    <!-- End Settings template -->

    {{> yield region="bodyChild" }}

</template>

The user layout is defined in imports/ui/components/user/user.layout.html

<template name="userLayout">
    <div class="layout-container">
        <div class="page-container bg-blue-grey-900">
            {{> yield }}
        </div>
    </div>
</template>

Usage

As mentioned, the entry of the application is the file client/main.js this file only imports the startup code listed in imports/startup/client/index.js

client/main.js

import '/imports/startup/client';

Note that by default, index.js is no required to be explicitly declared in the import statement.

imports/startup/client/index.js

import './vendor.js';

import '../../ui/components/app.scss';
import '../../ui/components/app.js';

import './routes.js';

Reminder: everything under the imports directory is not available to the app unless you import it first.

Vendor

The startup file will import first all vendor assets listed in vendor.js. This file will import (or require) all styles and javascripts of third party assets used in the app. By adding or remove files here you can customize the necessary feature for your final app.

Styles

The file imports/ui/components/app.scss is the main file to import custom styles into the app. You can create a custom component and import here your new styles.

Scripts

In the same way like styles, the file imports/ui/components/app.js is used to import all scripts for the app, adding a custom component you can import here your custom scripts to make them they be available

All scripts that access to dom object must be executed using the onRendered of the template that is using. For example:

import { Template } from 'meteor/templating';

Template.layout.onRendered(() => {

    // dom ready here

});

Routes

Finally, the route.js will be imported in order to define routes. To define routes we use iron:router

Examples


// Single route example
Router.route('dashboard', function() {
    this.render('Dashboard');
});

// Group route example
Router.map(function() {
    this.route('chart-flot', function() {
        this.render('chartsFlot');
    });
    this.route('chart-radial', function() {
        this.render('chartsRadial');
    });
    this.route('chart-rickshaw', function() {
        this.render('chartsRickshaw');
    });
});

Different layouts

To use a different template layout for a view just setup the layout property when defining the route

Router.route('login', function() {
    this.render('login');
    this.layout('userLayout');
});

Ripple Effect

The ripple can be achieved on almost any component by adding the class .ripple

This class is handled by a script that display a simple animation in the middle of the element where it's attached.

If you want to customized the styles of scripts of this effect, you find all related files under components/ripple

Translation

The translation system uses jQuery Localize This modules simplifies the translation system by loading translate references from a JSON file and replacing the content where the reference has been used.

Examples

<small data-localize="dashboard.WELCOME"></small>

The JSON files with translation references are located in the folder app/i18n

Example content of the file app/i18n/en.json

{
  "WELCOME": "Welcome",
  "header": {
    "SEARCH": "Search.."
  }
}

As a convention, we use the key name with the translated text in uppercase.

Settings and Themes

All settings related files belongs to components/settings

There you will find the initial values for the different settings supported. The template markup for the settings panel (the right sidebar) and the styles for such panel.

Themes

Theming is based on class-names that provides colors to components.

All themes classes available are configured in settings.js

var themes = [
    'theme-1',
    'theme-2',
    'theme-3',
    'theme-4',
    'theme-5',
    'theme-6',
    'theme-7',
    'theme-8',
    'theme-9',
]

Each of this theme, named theme-* is applied to the body tag via JavaScript.

You can find the source of the themes definition in file components/common/themes.scss

Default theme

The default is added by using the following code (meteor doesn't allow to change the body tag directly)

// /meteor/imports/ui/components/core/core.js
Template.layout.onRendered(() => {

    // Set default theme
    $('body').addClass('theme-1');

});
